# Source Policy

Public posts, articles, papers, and repositories are source leads, not benchmark ground truth.

Rules:

- Store source URLs and metadata.
- Do not republish private information.
- Do not republish screenshots without permission.
- For user-provided screenshots without a public URL, store only paraphrased source metadata and cleaned benchmark variants.
- Reproduce prompts independently.
- Prefer generated variants over copied viral prompts.
- Keep attribution for public archetypes where useful.
- Avoid naming private individuals in public artifacts.

Pre-publication checklist:

- No credentials, API keys, auth codes, or environment variables.
- No private account IDs, private emails, balances, or personal records.
- No copied screenshots without recorded permission.
- No raw provider request headers.
- No source text beyond short prompt/output snippets needed for reproducibility.
