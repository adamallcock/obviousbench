`report-ci-v0` means confidence intervals, not continuous integration. CI workflow changes are out of scope for this plan.

## Principles

- Point estimates alone are not enough for 40, 80, or 160 item panels.
- Use transparent formulas that do not require SciPy or pandas.
- Preserve current CSV compatibility by adding columns, not replacing existing ones.
- Prefer paired deltas on matched sample IDs over independent aggregate comparisons.

## Statistical Contract

### Wilson Interval

Use a 95 percent Wilson score interval for binomial accuracy-like metrics.

Inputs:

- `successes`: integer count,
- `n`: integer scored sample count,
- `z`: default `1.959963984540054`.

Outputs:

- `(lower, upper)` floats in `[0.0, 1.0]`.

For `n == 0`, return `(0.0, 0.0)`.

### Paired Delta

For two runs over matched `sample_id` values:

- `a_correct`: boolean from baseline,
- `b_correct`: boolean from comparison,
- `delta = mean(b_correct - a_correct)` over matched samples.

Also report:

- matched sample count,
- samples only in baseline,
- samples only in comparison,
- wins for comparison,
- losses for comparison,
- ties.

### Bootstrap Delta

For v0, bootstrap only paired deltas with deterministic seed `20260531` and `1_000` resamples. This is enough for local report uncertainty without adding dependencies.

## Functional Requirements

- `summary.csv` includes `accuracy_ci_low`, `accuracy_ci_high`, `answer_accuracy_ci_low`, `answer_accuracy_ci_high`, `strict_accuracy_ci_low`, and `strict_accuracy_ci_high`.
- `comparison.csv` preserves those columns.
- `delta.csv` includes paired sample counts and paired accuracy deltas when `usage_by_sample.csv` is available.
- `delta.csv` falls back to aggregate deltas only when per-sample rows are absent and marks `delta_method=aggregate_unpaired`.
- HTML report displays intervals for top-level accuracy and a caution when two runs differ by less than the larger interval half-width.
- No provider calls or external packages are required.

## Technical Requirements

- Round CSV floats to 6 decimal places for readability.
- Keep internal computations as floats until final projection.
- Do not make bootstrap run during `obviousbench summarize`; do it in `obviousbench build-comparison`, where multiple summaries are known.
- Keep bootstrap seed fixed unless a future CLI flag is added.

- [x] **Step 2: Implement Wilson helper**

Use the standard Wilson formula:

```python
def wilson_interval(successes: int, n: int, z: float = 1.959963984540054) -> tuple[float, float]:
    if n <= 0:
        return (0.0, 0.0)
    p = successes / n
    denominator = 1 + z * z / n
    center = (p + z * z / (2 * n)) / denominator
    margin = z * ((p * (1 - p) / n + z * z / (4 * n * n)) ** 0.5) / denominator
    return (max(0.0, center - margin), min(1.0, center + margin))
```
